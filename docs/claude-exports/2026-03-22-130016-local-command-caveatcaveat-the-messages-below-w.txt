     Plan: Safe Delete with Backup/Restore for Tasks, Pipelines, Sub-Agents

     Context

     Currently the system has delete only for pipelines and sub-agents (simple
     removal without backup or dependency checks). Tasks have no delete at all.
     The user needs:
     - Dependency-aware deletion (prevent breaking referential integrity)
     - Cascade delete for pipelines (exclusive tasks/agents deleted together)
     - Backup before any deletion to data/backup/ with SQLite log
     - Restore capability from backup
     - UI confirmation with dependency info before delete

     Architecture

     Three new backend packages + enhanced API handlers + frontend changes.

     ---
     Phase 1: Backend Foundation

     1.1 New package internal/depcheck/checker.go

     Pure-function dependency analyzer ‚ÄĒ no I/O, operates on *config.Config.

     type DeleteAnalysis struct {
         Entity       Dependency   // what is being deleted
         UsedBy       []Dependency // who references it
         CanDelete    bool
         CascadeItems []Dependency // exclusive items that will also be deleted
         Blocked      bool
         BlockReason  string
     }

     Functions:
     - AnalyzeTaskDelete(cfg, taskName) ‚ÄĒ blocked if any pipeline references the
      task
     - AnalyzePipelineDelete(cfg, pipelineName) ‚ÄĒ cascade-deletes tasks
     exclusive to this pipeline + their exclusive agents
     - AnalyzeSubAgentDelete(cfg, agentName) ‚ÄĒ blocked if any task references
     the agent
     - Helper: findPipelinesUsingTask, findTasksUsingAgent,
     isTaskExclusiveToPipeline

     Files: internal/depcheck/checker.go, internal/depcheck/checker_test.go

     1.2 New package internal/backup/manager.go

     Backup manager using the app's SQLite DB + file copies under
     data/backup/{id}/.

     New migration table backup_log:
     CREATE TABLE IF NOT EXISTS backup_log (
         id           TEXT PRIMARY KEY,
         entity_type  TEXT NOT NULL,
         entity_name  TEXT NOT NULL,
         action       TEXT NOT NULL,
         parent_id    TEXT NOT NULL DEFAULT '',
         config_snap  TEXT NOT NULL DEFAULT '',
         files        TEXT NOT NULL DEFAULT '',
         created_at   DATETIME NOT NULL,
         restored_at  DATETIME
     )

     Manager methods:
     - CreateBackup(ctx, entry BackupEntry, filesToCopy map[string]string) ‚ÄĒ
     inserts row, copies files to backup dir
     - ListBackups(ctx, filter) ‚ÄĒ list with optional entity_type filter
     - GetBackup(ctx, id) ‚ÄĒ single entry + backed-up file paths
     - RestoreFiles(ctx, id) ‚ÄĒ returns config snapshot + file contents from
     backup dir
     - MarkRestored(ctx, id) ‚ÄĒ sets restored_at

     Key design:
     - config_snap stores full tasks.yaml content at time of backup (safe
     restore via re-parse + validate)
     - Sub-agent .md files copied to data/backup/{id}/agents/
     - Domain data dirs are NOT backed up (too large); only config entry +
     DOMAIN.md backed up
     - Cascade delete creates one parent backup + child entries linked via
     parent_id

     Files: internal/backup/manager.go, internal/backup/manager_test.go,
     migration in internal/store/sqlite/migrations.go

     1.3 Domain Manager: add RemoveDomain method

     In internal/domain/manager.go, add:
     func (m *Manager) RemoveDomain(name string) { delete(m.domains, name) }
     Domain data dir is NOT deleted (left on disk as safety measure). Only
     removed from in-memory map + config.

     Phase 2: API Handlers

     2.1 Wire backupMgr into Server

     - Add backupMgr *backup.Manager field to Server struct in router.go
     - Pass DB handle from cmd/server/main.go to create backup manager
     - Add to NewServer() params

     2.2 Delete-info endpoints (new file internal/api/delete.go)

     Pre-delete analysis ‚ÄĒ frontend calls before showing confirmation:

     GET /api/v1/tasks/{name}/delete-info      ‚Üí DeleteAnalysis JSON
     GET /api/v1/pipelines/{name}/delete-info  ‚Üí DeleteAnalysis JSON
     GET /api/v1/subagents/{name}/delete-info  ‚Üí DeleteAnalysis JSON

     2.3 Task delete (new, in internal/api/tasks.go)

     DELETE /api/v1/tasks/{name}

     1. depcheck.AnalyzeTaskDelete() ‚ÄĒ if blocked, return 409 with analysis
     2. Read current tasks.yaml for backup
     3. backupMgr.CreateBackup() with config snapshot
     4. Remove task from cfg.Tasks
     5. Clean task name from domain Tasks lists
     6. config.Validate() ‚ÄĒ if fails, rollback in-memory, return 500
     7. cfg.Save()
     8. Return 200 {backup_id, deleted: [...]}

     2.4 Enhanced pipeline delete (internal/api/pipelines.go)

     DELETE /api/v1/pipelines/{name}

     1. depcheck.AnalyzePipelineDelete() ‚ÄĒ get cascade items
     2. Backup config + cascade agent .md files
     3. Delete cascade sub-agents via subagentMgr.Delete()
     4. Remove cascade tasks from cfg.Tasks
     5. Remove pipeline from cfg.Pipelines
     6. Clean domain references
     7. Validate + save
     8. Return 200 with backup_id + all deleted items

     2.5 Enhanced sub-agent delete (internal/api/subagents.go)

     1. depcheck.AnalyzeSubAgentDelete() ‚ÄĒ if blocked, return 409
     2. Backup .md file
     3. subagentMgr.Delete()
     4. Return 200 with backup_id

     2.6 Backup/Restore endpoints (new file internal/api/backups.go)

     GET  /api/v1/backups              ‚Üí list backups (most recent first)
     GET  /api/v1/backups/{id}         ‚Üí single backup details
     POST /api/v1/backups/{id}/restore ‚Üí restore from backup

     Restore logic:
     1. Load backup entry + config snapshot
     2. Check name conflicts ‚ÄĒ if entity with same name exists, return 409
     3. Parse backed-up YAML, extract relevant entity, merge into current config
     4. For sub-agents, write .md file back from backup
     5. config.Validate() merged config
     6. cfg.Save()
     7. Mark backup as restored

     2.7 Concurrency safety

     Use s.guard.TryAcquire("config:write") / Release in all delete + restore
     handlers to serialize config modifications.

     2.8 Route registration in router.go

     // Task delete

     s.withAuth(s.handlePipelineDeleteInfo))

     s.withAuth(s.handleSubAgentDeleteInfo))

     // Backups

     mux.HandleFunc("GET /api/v1/backups/{id}", s.withAuth(s.handleGetBackup))
     mux.HandleFunc("POST /api/v1/backups/{id}/restore",
     s.withAuth(s.handleRestoreBackup))
