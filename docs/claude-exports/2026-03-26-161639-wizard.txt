     Wizard Troubleshooter — план реализации

     Context

     Wizard часто генерирует ошибки, пустые результаты и невалидные
     конфигурации. Сейчас при ошибке пользователь видит только generic сообщение
      и должен вручную дебажить через CLI. Нужен механизм диагностики, retry с
     контекстом, редактирование плана и тестовый запуск — всё из UI.

     Архитектура решения

     Три новых возможности:
     1. Structured diagnosis — категоризация ошибок + предложения по исправлению
     2. Retry с контекстом — повторная генерация с информацией о предыдущей
     ошибке
     3. Test run — запуск первой задачи после apply для проверки через
     outputcheck

     UI расширяется с 3 до 5 состояний: input → preview → editing → result → 
     testing

     Шаг 2: Диагностика ошибок — internal/wizard/troubleshoot.go (новый файл)

     Функция DiagnoseError(phase string, err error, rawOutput string, plan 
     *Plan) *WizardDiagnosis:

     Фаза generate:
     - "empty output" → ErrCatEmptyOutput, suggestions: retry, retry_simplified
     - "parsing plan JSON" → ErrCatJSONParse, details = rawOutput, suggestions:
     retry, edit_raw
     - context.DeadlineExceeded → ErrCatTimeout, suggestions: retry, simplify

     Фаза validate:
     - "already exists" → ErrCatDuplicateName, вычислить PatchedPlan с суффиксом
      -v2
     - "references unknown" → ErrCatMissingReference, suggestions: edit_plan,
     retry
     - "permission_mode" → ErrCatPermissionMode, suggestions: auto_fix (уже есть
      в validate)

     Фаза apply:
     - Любая ошибка → ErrCatApplyFailed, suggestions: edit_plan, retry

     Фаза test:
     - soft failure → ErrCatTestSoftFailure, suggestions: edit_task, add_tools,
     regenerate

     Вспомогательная функция AutoFixDuplicateNames(plan *Plan, cfg 
     *config.Config) *Plan:
     - Находит конфликтующие имена
     - Добавляет суффикс -v2 (или -v3 и т.д.)
     - Обновляет ВСЕ ссылки: pipeline steps, domain lists, task agents
