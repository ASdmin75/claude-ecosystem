  1. Claude Code CLI в headless/SDK режиме

  Ключевой инструмент — claude CLI можно вызывать программно:

  # Одноразовый запрос (pipe mode)
  echo "Напиши unit тест для функции ParseConfig" | claude -p

  # С указанием файлов контекста
  claude -p "Отрефактори этот код" < main.go

  # JSON output для парсинга в Go
  claude -p --output-format json "Объясни ошибку" < error.log

  2. Hooks (события)

  Claude Code поддерживает hooks — shell-команды, которые выполняются при
  определённых событиях. Настраиваются в .claude/settings.json или в проектном
  CLAUDE.md:

  // .claude/settings.json
  {
    "hooks": {
      "PreToolUse": [
        {
          "matcher": "Bash",
          "command": "your-go-binary validate-command"
        }
      ],
      "PostToolUse": [
        {
          "matcher": "Edit",
          "command": "your-go-binary post-edit-action"
        }
      ],
      "Notification": [
        {
          "command": "your-go-binary on-notification"
        }
      ]
    }
  }

  Типы хуков: PreToolUse, PostToolUse, Notification, Stop.

  3. Sub-agents через CLI

  Claude Code имеет встроенный Agent tool — но для вашей экосистемы на Go
  интереснее вызывать claude как субпроцесс:

  // Go-оркестратор, запускающий Claude как sub-agent
  func runClaudeAgent(ctx context.Context, prompt string, workDir string) 
  (string, error) {
      cmd := exec.CommandContext(ctx, "claude", "-p", "--output-format", "json")
      cmd.Dir = workDir
      cmd.Stdin = strings.NewReader(prompt)

      output, err := cmd.Output()
      if err != nil {
          return "", fmt.Errorf("claude agent failed: %w", err)
      }
      return string(output), nil
  }

  Ключевые компоненты

  1. Scheduler (по расписанию)
  // Используем robfig/cron для запуска агентов по расписанию
  c := cron.New()
  c.AddFunc("0 9 * * *", func() {
      // Утренний code review новых PR
      runClaudeAgent(ctx, "Review all open PRs in ./src", projectDir)
  })
  c.AddFunc("*/30 * * * *", func() {
      // Каждые 30 мин проверяем TODO
      runClaudeAgent(ctx, "Find and prioritize TODOs in codebase", projectDir)
  })

  2. Watcher (по событиям файловой системы)
  // fsnotify для реакции на изменения файлов
  watcher, _ := fsnotify.NewWatcher()
  watcher.Add("./src")

  for event := range watcher.Events {
      if event.Op&fsnotify.Write != 0 && strings.HasSuffix(event.Name, ".go") {
          // Автоматический review при сохранении .go файла
          go runClaudeAgent(ctx,
              fmt.Sprintf("Review changes in %s, suggest improvements",
  event.Name),
              projectDir)
      }
  }

  3. Hook-интеграция (Claude вызывает ваш Go-бинарник)
  // Ваш Go-бинарник как hook для Claude Code
  // Claude передаёт контекст через stdin в JSON
  func main() {
      var hookData struct {
          Tool   string `json:"tool_name"`
          Input  any    `json:"tool_input"`
      }
      json.NewDecoder(os.Stdin).Decode(&hookData)

      // Логика: валидация, логирование, запуск другого агента
      if hookData.Tool == "Bash" {
          // Например, проверить команду перед выполнением
      }
  }

  4. Multi-session через --session-id и --resume
  # Создать именованную сессию
  claude -p --session-id "code-review-agent" "Start reviewing PR #42"

  # Продолжить в той же сессии (сохраняет контекст)
  claude -p --resume "code-review-agent" "Now check the test coverage"

  Практический план действий

  1. Создать Go-проект оркестратора — он будет точкой входа для всей экосистемы
  2. Определить агентов как конфиги (prompt template + working dir +
  schedule/trigger)
  3. Использовать claude -p для каждого агента как subprocess
  4. Hooks — зарегистрировать Go-бинарник как hook в .claude/settings.json для
  реакции на действия Claude
  5. CLAUDE.md в каждом проекте — инструкции для агента, специфичные для
  контекста

  Как использовать

  # Собрать
  make build

  # Запустить конкретного агента разово
  make run-agent AGENT=code-review

  # Запустить демон (scheduler + watcher)
  make run

  # Установить hook-бинарник
  make install

  Ключевые идеи

  - claude -p --output-format json — основной способ взаимодействия, никакого
  OAuth не нужно
  - Scheduler (robfig/cron) — запускает агентов по cron-расписанию
  - Watcher (fsnotify) — запускает агентов по событиям файловой системы с
  debounce
  - Event bus — агенты могут реагировать на результаты друг друга через
  bus.Subscribe
  - Hook binary — Claude Code вызывает ваш Go-бинарник перед/после каждого
  инструмента, можно валидировать, логировать, запускать цепочки
  - Template variables — промпты поддерживают {{.File}} и другие переменные из
  контекста события

  Всё работает через локальный CLI без внешних API или OAuth.
