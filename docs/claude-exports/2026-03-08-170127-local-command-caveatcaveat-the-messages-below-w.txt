  Создан systemd user service

  deploy/claude-ecosystem.service — unit-файл для systemd user daemon:
  - Type=simple — сервер работает на переднем плане
  - WorkingDirectory — корень проекта (чтобы tasks.yaml, .claude/agents/ и
  MCP-бинарники в bin/ были доступны)
  - EnvironmentFile — загружает .env (с - — не падает если файла нет)
  - Environment=PATH= — включает ~/.local/bin, чтобы claude CLI был доступен
  - Restart=on-failure с 5-секундной паузой
  - KillSignal=SIGTERM — сервер уже корректно обрабатывает graceful shutdown
  - Логи через journalctl

  Новые Makefile targets

  ┌────────────────────┬────────────────────────────────────────────────────┐
  │       Target       │                      Описание                      │
  ├────────────────────┼────────────────────────────────────────────────────┤
  │ make               │ Собирает бинарники, копирует service в             │
  │ daemon-install     │ ~/.config/systemd/user/, включает автозапуск       │
  ├────────────────────┼────────────────────────────────────────────────────┤
  │ make               │ Останавливает, отключает и удаляет service         │
  │ daemon-uninstall   │                                                    │
  ├────────────────────┼────────────────────────────────────────────────────┤
  │ make daemon-start  │ Запускает демон                                    │
  ├────────────────────┼────────────────────────────────────────────────────┤
  │ make daemon-stop   │ Останавливает демон                                │
  ├────────────────────┼────────────────────────────────────────────────────┤
  │ make               │ Перезапускает демон                                │
  │ daemon-restart     │                                                    │
  ├────────────────────┼────────────────────────────────────────────────────┤
  │ make daemon-status │ Показывает статус                                  │
  ├────────────────────┼────────────────────────────────────────────────────┤
  │ make daemon-logs   │ Показывает логи в реальном времени (-f)            │
  └────────────────────┴────────────────────────────────────────────────────┘

  Типичный workflow

  # Первый запуск
  make daemon-install
  make daemon-start

  # Посмотреть статус и логи
  make daemon-status
  make daemon-logs

  # После обновления кода
  make build
  make daemon-restart

  # Удалить
  make daemon-uninstall

  Используется user-level systemd (systemctl --user), поэтому sudo не нужен —
  демон работает от текущего пользователя и имеет доступ к его claude
  credentials.
