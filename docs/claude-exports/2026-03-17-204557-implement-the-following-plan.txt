│ Implementation structure (single main.go)                                    │
│                                                                              │
│ 1. JSON-RPC boilerplate (~50 lines)                                          │
│                                                                              │
│ Копируем паттерн из mcp-database/main.go: structs jsonRPCRequest,            │
│ jsonRPCResponse, tool, toolCallParams, contentItem, toolResult, helpers      │
│ textResult/errorResult, main loop с initialize/tools/list/tools/call.        │
│                                                                              │
│ 2. Startup: parse spec & generate tools                                      │
│                                                                              │
│ main():                                                                      │
│   specPath = os.Getenv("OPENAPI_SPEC_PATH")  // required                     │
│   specData = os.ReadFile(specPath)                                           │
│   doc = libopenapi.NewDocument(specData)                                     │
│   model = doc.BuildV3Model() // or BuildV2Model() based on version           │
│   operations = extractOperations(model)  // -> []apiOperation                │
│   operations = filterOperations(operations)  // by tags/paths/ops            │
│   generatedTools = generateTools(operations)  // -> map[string]generatedTool │
│   // start JSON-RPC loop with generatedTools                                 │
│                                                                              │
│ 3. Operation extraction                                                      │
│                                                                              │
│ Walk model.Paths → for each path + method → normalize into:                  │
│ type apiOperation struct {                                                   │
│     OperationID string                                                       │
│     Method, Path, Summary, Description string                                │
│     Tags        []string                                                     │
│     Parameters  []apiParameter   // path, query, header                      │
│     RequestBody *apiRequestBody  // body schema for POST/PUT/PATCH           │
│ }                                                                            │
│ Поддержка v2 и v3 через отдельные extractV2/extractV3 функции.               │
│                                                                              │
│ 4. Tool name generation                                                      │
│                                                                              │
│ - Если operationId задан: sanitize(operationId) → lowercase, non-alnum → _   │
│ - Иначе: method_pathToSnake(path) → get_users_by_id_orders                   │
│ - Дедупликация через суффиксы _2, _3                                         │
│                                                                              │
│ 5. Tool InputSchema generation                                               │
│                                                                              │
│ Для каждой операции:                                                         │
│ - Path params → top-level required string properties                         │
│ - Query params → top-level properties (required if marked)                   │
│ - Header params → properties с префиксом header_                             │
│ - Request body → property body с вложенной JSON Schema из спеки (ограничение │
│  глубины до 3 уровней для $ref)                                              │
│                                                                              │
│ 6. Tool execution (tools/call handler)                                       │
│                                                                              │
│ handleToolCall(toolName, arguments):                                         │
│   op = generatedTools[toolName].operation                                    │
│   1. URL: baseURL + op.Path, подставить path params из arguments             │
│   2. Query string: из query params в arguments                               │
│   3. Headers: Content-Type + auth + header params + extra headers            │
│   4. Body: marshal arguments["body"] в JSON (для POST/PUT/PATCH)             │
│   5. HTTP request с timeout                                                  │
│   6. Response: read body (limit 1MB), return "HTTP {status}\n\n{body}"       │
│      - JSON → pretty-print                                                   │
│      - status >= 400 → isError: true                                         │
│                                                                              │
│ 7. Auth                                                                      │
│                                                                              │
│ В applyAuth(req): switch на OPENAPI_AUTH_TYPE:                               │
│ - bearer → Authorization: Bearer {OPENAPI_AUTH_TOKEN}                        │
│ - apikey → header или query param по OPENAPI_API_KEY_NAME/OPENAPI_API_KEY_IN │
│ - basic → Authorization: Basic base64(user:pass)                             │

│ Design decisions                                                             │
│                                                                              │
│ - libopenapi (pb33f) — поддержка v2+v3+v3.1, разрешение $ref, активно        │
│ поддерживается                                                               │
│ - Max 50 tools — большие спеки деградируют качество tool selection;          │
│ фильтрация через теги/пути решает проблему                                   │
│ - Без OAuth2 flow — bearer token покрывает большинство случаев, пользователь │
│  получает токен внешне                                                       │
│ - Без file upload — multipart/binary скипаются (непрактично для LLM)         │
│ - Response limit 1MB — защита от OOM                                         │
│ - Schema depth limit 3 — для circular $ref в InputSchema                     │
