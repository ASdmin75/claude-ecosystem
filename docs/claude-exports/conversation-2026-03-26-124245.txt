     Demo: AviationStack API через mcp-openapi

     Context

     Завтра (2026-03-24) показ руководству логистической компании (авиаперевозки
      + мультимодал). Цель — продемонстрировать потенциал Claude Ecosystem для
     автоматизации бизнес-процессов. Ключевой фокус: mcp-openapi — подключение к
      ЛЮБОМУ API без написания кода, только через OpenAPI-спецификацию.

     Ограничения AviationStack Free:
     - 100 запросов/месяц, HTTP only (не HTTPS!)
     - Доступные эндпоинты: /flights, /timetable, /flightsFuture, /airports
     - Недоступно: airlines, aircraft, routes, cities, countries
     - Rate limit на timetable/flightsFuture: 1 req/60s
     - Auth: API key как query parameter access_key

     Бюджет запросов на демо: ~15-20 из 100 (тестирование + показ)

     ---
     Шаг 1: Создать OpenAPI-спецификацию для AviationStack

     Файл: specs/aviationstack.yaml

     AviationStack не предоставляет готовую спецификацию — создаём вручную на
     основе документации. Включаем только доступные на free-тарифе эндпоинты:

     - GET /v1/flights — реальные рейсы (фильтры: airline_iata, flight_iata,
     dep_iata, arr_iata, flight_status, flight_date, limit, offset)
     - GET /v1/airports — справочник аэропортов (search по IATA-коду)
     - GET /v1/timetable — расписание на сегодня (iata_code, type:
     departure/arrival)
     - GET /v1/flightsFuture — будущие рейсы (iata_code, type, date)

     Base URL: http://api.aviationstack.com (HTTP, не HTTPS!)

     Auth: apikey в query param access_key — настроим через OPENAPI_API_KEY_IN: 
     query, OPENAPI_API_KEY_NAME: access_key.

     ---
     Шаг 2: Настроить MCP-сервер в tasks.yaml

     Добавить в секцию mcp_servers:

     - name: aviationstack
       command: ./bin/mcp-openapi
       env:
         OPENAPI_SPEC_PATH: specs/aviationstack.yaml
         OPENAPI_BASE_URL: http://api.aviationstack.com
         OPENAPI_AUTH_TYPE: apikey
         OPENAPI_API_KEY: ${AVIATIONSTACK_API_KEY}
         OPENAPI_API_KEY_NAME: access_key
         OPENAPI_API_KEY_IN: query

     Добавить AVIATIONSTACK_API_KEY в .env.

     ---
     Шаг 3: Создать домен aviation-monitoring

     Директория данных: data/aviation-monitoring/
     БД: flights.db

     Таблицы:
     CREATE TABLE IF NOT EXISTS tracked_flights (
         id INTEGER PRIMARY KEY AUTOINCREMENT,
         flight_iata TEXT NOT NULL,
         airline_name TEXT,
         dep_iata TEXT,
         arr_iata TEXT,
         flight_date TEXT,
         status TEXT,
         dep_scheduled TEXT,
         dep_actual TEXT,
         arr_scheduled TEXT,
         arr_actual TEXT,
         delay_minutes INTEGER DEFAULT 0,
         checked_at TEXT DEFAULT (datetime('now')),
         raw_json TEXT
     );

     CREATE TABLE IF NOT EXISTS hub_snapshots (
         id INTEGER PRIMARY KEY AUTOINCREMENT,
         hub_iata TEXT NOT NULL,
         snapshot_date TEXT NOT NULL,
         total_flights INTEGER,
         cargo_flights INTEGER,
         delayed_flights INTEGER,
         cancelled_flights INTEGER,
         report_text TEXT,
         created_at TEXT DEFAULT (datetime('now'))
     );

     DOMAIN.md — инструкции для агента: какие таблицы есть, как записывать
     данные, примеры использования MCP-инструментов.

     ---
     Шаг 4: Создать задачи

     4.1 aviation-flight-tracker

     Назначение: Проверяет статус грузовых рейсов на заданном маршруте,
     сохраняет в БД.

     Промпт: Найди текущие грузовые рейсы из {{dep}} в {{arr}} используя
     AviationStack API.
     Для каждого найденного рейса сохрани данные в таблицу tracked_flights.
     Выведи краткую сводку: количество рейсов, задержки, отмены.

     MCP серверы: aviationstack, database
     Allowed tools: mcp__aviationstack__*, mcp__database__*

     4.2 aviation-hub-monitor

     Назначение: Мониторинг грузового хаба — вылеты и прилёты за сегодня.

     Промпт: Проверь расписание аэропорта {{hub_iata}} (вылеты и прилёты) на
     сегодня.
     Выдели грузовые рейсы (cargo). Посчитай статистику.
     Сохрани снимок в hub_snapshots. Выведи отчёт.

     MCP серверы: aviationstack, database

     4.3 aviation-compile-report

     Назначение: Собирает Excel-отчёт из данных БД.

     Промпт: На основе данных из предыдущего шага ({{.PrevOutput}})
     создай Excel-отчёт с рейсами, задержками и статистикой.

     MCP серверы: database, excel, filesystem

     4.4 aviation-deliver-report

     Назначение: Отправляет отчёт в Telegram.

     Промпт: Отправь отчёт из предыдущего шага в Telegram.
     Приложи Excel-файл если он есть.

     MCP серверы: telegram, filesystem

     ---
     Шаг 5: Создать пайплайн

     aviation-cargo-monitor

     mode: sequential
     steps:
       - task: aviation-hub-monitor
       - task: aviation-compile-report
       - task: aviation-deliver-report
     max_iterations: 1

     ---
     Шаг 6: Сценарий демонстрации

     Акт 1: Wizard (создание через UI)

     Показать как через Wizard в Web UI можно создать новую задачу/пайплайн —
     подключить API без единой строчки кода.

     Акт 2: CLI — единичный запрос

     make run-task TASK=aviation-flight-tracker
     Claude сам находит нужный MCP-инструмент, вызывает API, парсит ответ,
     сохраняет в БД, выдаёт сводку. ~2-3 API-запроса.

     Акт 3: Web UI — полный пайплайн

     Запустить aviation-cargo-monitor через Web UI. Руководство видит:
     1. SSE-стриминг выполнения в реальном времени
     2. Claude мониторит хаб → компилирует Excel → отправляет в Telegram
     3. Сообщение приходит в Telegram-чат прямо на экране

     ~3-5 API-запросов.

     Ключевые тезисы для руководства:

     - "Мы не написали ни строчки кода для интеграции с AviationStack"
     - "Достаточно OpenAPI-спецификации — Claude сам понимает как использовать
     API"
     - "Любой API с документацией можно подключить за 10 минут"
     - "Пайплайны автоматизируют цепочку: мониторинг → анализ → отчёт →
     доставка"
