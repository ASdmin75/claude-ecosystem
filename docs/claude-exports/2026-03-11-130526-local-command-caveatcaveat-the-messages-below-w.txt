● This is a known behavior of Claude Code CLI. When claude -p runs, it may
  trigger GitHub OAuth if:

  1. No cached auth token — Claude Code stores its auth in ~/.claude/ config. If
   the token expires or isn't persisted between runs, it re-authenticates each
  time.
  2. Spawning many parallel processes — Your system runs claude -p as
  subprocesses. If multiple instances start simultaneously before auth is
  cached, they can each trigger auth independently.

  Claude Code authenticates via claude.ai OAuth, not GitHub. If you're seeing a
  browser popup each time claude -p runs, there are a few possible causes:

  1. Subprocess can't read cached credentials — when your Go server spawns
  claude -p, the subprocess might run with a different HOME or user context.
  Check:

  # What HOME does the subprocess see?
  claude -p "echo $HOME" --print

  2. Token refresh issue — your token expires at a specific time, and if the
  refresh flow fails silently, it re-triggers full auth.
  3. MCP server triggering GitHub auth — if any of your MCP servers or tasks use
   GitHub tools, those might prompt for GitHub OAuth separately.

  To fix it for non-interactive claude -p usage, you can set an API key instead
  of relying on OAuth:

  export ANTHROPIC_API_KEY="sk-ant-..."

  Then claude -p will use the API key directly and skip OAuth entirely. You can
  set this in your tasks.yaml or in the environment where your server runs.

● Вот и причина! Две проблемы:

  1. Аккаунт AlexStav — невалидный токен. GitHub CLI (gh) имеет два аккаунта, и
  токен для AlexStav протух. При каждом запуске Claude Code делает git fetch
  маркетплейса плагинов с GitHub, и это может триггерить диалог авторизации.
  2. Git operations protocol: ssh, но маркетплейс клонирован через https — это
  несовпадение тоже может вызывать запрос credentials.

  Решение — уберите сломанный аккаунт:

  gh auth logout -h github.com -u AlexStav

  А если диалог всё равно будет появляться, можно отключить плагин gopls-lsp
  (единственный установленный), чтобы Claude Code не обновлял маркетплейс при
  запуске. Для этого в ~/.claude/settings.json уберите:

  "enabledPlugins": {
      "gopls-lsp@claude-plugins-official": true
  }
