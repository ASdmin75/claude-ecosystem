  New field allow_concurrent on both Task and Pipeline config structs (*bool,

  New package internal/runguard/ тАФ a simple mutex-based guard that tracks which
  tasks/pipelines are currently running. Shared across all execution paths (API,
   scheduler, watcher).

  How it works

  When allow_concurrent: false is set on a task or pipeline:
  - Scheduler (cron): skips the run with a log message if already running
  - API (manual /run and /run-async): returns 409 Conflict with "task/pipeline 
  is already running"
  - Watcher (file change): skips the run with a log message if already running
  - RunPipelineByName (scheduler pipeline callback): skips with a log message

  The guard uses a "task:<name>" / "pipeline:<name>" key namespace to avoid
  collisions between tasks and pipelines with the same name.

  Usage in tasks.yaml

      prompt: "..."
      allow_concurrent: false   # prevent parallel runs
      schedule: "*/5 * * * *"
