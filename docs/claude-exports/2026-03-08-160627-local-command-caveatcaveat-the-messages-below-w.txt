● MCP серверы (excel, email, telegram, filesystem) определены в tasks.yaml и
  запускаются внутри основного процесса server через mcpmanager — как дочерние
  процессы (stdio JSON-RPC). Они не являются отдельными сетевыми сервисами,
  поэтому им не нужны отдельные контейнеры в docker-compose.yml.

  Бинарники MCP серверов (./bin/mcp-*) собираются при make build и копируются в
  Docker-образ через Dockerfile. Сервер запускает их по мере необходимости.

  Хотите вынести MCP серверы в отдельные контейнеры, или текущая архитектура
  (дочерние процессы внутри одного контейнера) вас устраивает?

● Итого, что сделано:

  1. Убрал :ro с tasks.yaml и .claude/agents/ — теперь UI может сохранять
  изменения.

  2. Добавил volume workspace в docker-compose:
  - ${WORKSPACE_DIR:-./workspace}:/app/workspace

  3. Обновил пути в задачах:
  - Excel-отчёт → workspace/leads/leads-report-{{.Date}}.xlsx
  - Копия для CEO → workspace/ceo/reports/

  4. Добавил WORKSPACE_DIR в .env для настройки пути на хосте.

  Структура:
  workspace/
  ├── leads/          # отчёты по лидам
  ├── ceo/reports/    # копии для CEO
  └── .gitignore      # данные не попадают в git

  Задачи работают с work_dir: . (корень /app), поэтому относительный путь
  workspace/... маппится в /app/workspace/... внутри контейнера, что
  соответствует смонтированному volume. При необходимости можно добавлять новые
  подкаталоги для других задач.
