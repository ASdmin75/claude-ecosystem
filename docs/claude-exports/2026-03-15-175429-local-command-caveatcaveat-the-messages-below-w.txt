● Fix applied. Here's what was wrong:

  Root cause: When a pipeline runs via cron, the browser tab is typically in the
   background. Browsers throttle/suspend background tabs, causing the SSE
  (EventSource) connection to drop. Events published during the disconnection
  (like pipeline.started, pipeline.completed) are lost because the event bus has
   no replay mechanism. When the user returns to the tab, the connection
  reconnects but the old onopen handler only reset the backoff delay — it never
  refreshed the data.

  Changes in useSSE.ts:
  1. Invalidate queries on SSE reconnect — when the connection drops and
  re-establishes (onopen after a prior connection), all execution/dashboard
  queries are refetched to catch missed events
  2. Invalidate queries on tab visibility change — when the user switches back
  to the tab, queries are refetched immediately via visibilitychange listener

● Супергруппа — это расширенный тип группы в Telegram с дополнительными
  возможностями:
  - До 200 000 участников (обычная — до 200)
  - История сообщений для новых участников
  - Закреплённые сообщения, админ-права, антиспам
  - Все публичные группы — супергруппы

  Расконвертировать нельзя — миграция в Telegram односторонняя и необратимая.

  Но это и не нужно — супергруппа работает так же как обычная, просто с другим
  chat_id. Мы его уже обновили, всё работает.

  Скорее всего миграция произошла не при добавлении бота, а позже — когда вы
  включили историю сообщений для новых участников или изменили настройки
  админов. Telegram делает это тихо, без уведомлений.
