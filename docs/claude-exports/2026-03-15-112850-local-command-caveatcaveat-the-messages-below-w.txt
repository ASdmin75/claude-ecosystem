‚óŹ I found the issue. The problem is in the useSSE hook: when the SSE connection
  drops (common when the browser tab is in the background waiting for cron) and
  reconnects, it doesn't invalidate queries. Events published during the
  disconnection gap are lost forever since there's no replay mechanism.

  The fix is to invalidate queries on reconnection (onopen) and when the tab
  becomes visible again.

‚óŹ Fix applied. Here's what was wrong:

  Root cause: When a pipeline runs via cron, the browser tab is typically in the
   background. Browsers throttle/suspend background tabs, causing the SSE
  (EventSource) connection to drop. Events published during the disconnection
  (like pipeline.started, pipeline.completed) are lost because the event bus has
   no replay mechanism. When the user returns to the tab, the connection
  reconnects but the old onopen handler only reset the backoff delay ‚ÄĒ it never
  refreshed the data.

  Changes in useSSE.ts:
  1. Invalidate queries on SSE reconnect ‚ÄĒ when the connection drops and
  re-establishes (onopen after a prior connection), all execution/dashboard
  queries are refetched to catch missed events
  2. Invalidate queries on tab visibility change ‚ÄĒ when the user switches back
  to the tab, queries are refetched immediately via visibilitychange listener
