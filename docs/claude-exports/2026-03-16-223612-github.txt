● Вот в чём проблема:

  На GitHub дефолтная ветка — main, но в ней только 1 коммит (Initial commit
  92189b2). Вся разработка (27 коммитов) ведётся в ветке master, которая
  полностью синхронизирована с origin/master.

  Когда вы открываете репозиторий на GitHub, он показывает ветку main с
  единственным начальным коммитом — поэтому кажется, что репо почти пустое или
  сильно отстаёт.

  Два варианта исправления:

  1. Сменить дефолтную ветку на GitHub на master — если хотите продолжать
  работать в master:
  gh repo edit ASdmin75/claude-ecosystem --default-branch master
  1. После этого можно удалить ветку main на GitHub.
  2. Слить master в main — если хотите перейти на main как основную:
  git checkout main && git merge master && git push origin main

● Вот что произошло:

  1. На GitHub репозиторий был создан через веб-интерфейс (кнопка "Create
  repository") — GitHub по умолчанию создаёт ветку main. Первый коммит 92189b2 
  Initial commit был сделан там.
  2. Локально репозиторий был инициализирован отдельно (git init), и у вас в
  системе не настроен init.defaultBranch — значит git использовал старое
  значение по умолчанию — master. Первый локальный коммит 6c0b57c feat: v2 
  architecture refactoring был сделан в master.
  3. Когда вы подключили remote и запушили, git отправил ветку master на GitHub.
   Но main на GitHub осталась нетронутой со своим единственным коммитом. Ветки
  main и master — это два независимых дерева с разными initial commits.

  Короче: GitHub создал main, а локальный git создал master. Вся работа пошла в
  master, а main на GitHub так и осталась с одним коммитом.

  Исправить проще всего — сменить дефолтную ветку на GitHub и удалить main:

  gh repo edit --default-branch master
  git push origin --delete main
