● Нет, Claude Code не требует GitHub для работы. Но он может пытаться обращаться
   к GitHub если:

  1. work_dir указывает на git-репозиторий с remote на GitHub
  2. Claude Code при запуске делает git fetch или проверяет remote

  Самое простое решение — передать GIT_TERMINAL_PROMPT=0 и GIT_SSH_COMMAND чтобы
   подавить любые SSH-запросы. Либо вообще отключить git-операции через
  переменную окружения.

  Давайте посмотрим, какие env vars передаются процессу и можно ли отключить
  SSH-промпты:

● Проблема ясна. cmd наследует окружение родительского процесса, но не
  устанавливает переменные для подавления интерактивных SSH-запросов. Когда
  claude -p запускается в git-репозитории, он может делать git fetch/git status
  с remote, что вызывает SSH-промпт.
