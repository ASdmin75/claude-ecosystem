│ Plan to implement                                                            │
│                                                                              │
│ Wizard: Auto-Generation of Tasks, Agents, Pipelines, Domains                 │
│                                                                              │
│ Context                                                                      │
│                                                                              │
│ Currently, creating a new automation in Claude Ecosystem requires manually   │
│ configuring multiple entities: tasks in tasks.yaml, sub-agent .md files,     │
│ domain schemas, pipelines — and understanding how they all wire together.    │
│ The wizard automates this: user describes what they want in natural          │
│ language, Claude analyzes it, and generates all necessary configuration      │
│ entities for optimal execution.                                              │
│                                                                              │
│ Architecture                                                                 │
│                                                                              │
│ Two-phase flow: Generate (Claude produces a structured plan via json_schema) │
│  → Preview/Edit → Apply (create all entities in dependency order).           │
│                                                                              │
│ The wizard doesn't need its own data model — it reuses existing config       │
│ structs. Plans are held in memory with TTL (no persistence needed for a      │
│ generate→preview→apply cycle).                                               │
│                                                                              │
│ Backend                                                                      │
│                                                                              │
│ New package: internal/wizard/                                                │

│ types.go — Plan and sub-plan structs:                                        │
│ - GenerateRequest{Description, WorkDir}                                      │
│ - Plan{ID, Description, Summary, Domains[], Agents[], Tasks[], Pipelines[],  │
│ Status}                                                                      │
│ - DomainPlan{Name, Description, DataDir, DB, Schema, DomainDoc, Tasks[],     │
│ Pipelines[], Agents[], MCPServers[]}                                         │
│ - AgentPlan{Name, Description, Tools[], Model, PermissionMode, Instructions, │
│  Scope}                                                                      │
│ - TaskPlan{Name, Prompt, WorkDir, Schedule, Tags[], Model, Timeout,          │
│ Agents[], MCPServers[], AllowedTools[], MaxTurns, MaxBudgetUSD,              │
│ PermissionMode, Domain}                                                      │
│ - PipelinePlan{Name, Mode, Steps[], MaxIterations, StopSignal}               │
│ - ApplyResult{DomainsCreated[], AgentsCreated[], TasksCreated[],             │
│ PipelinesCreated[], Errors[]}                                                │
│                                                                              │
│ prompt.go — Wizard prompt template + JSON schema constant:                   │
│ - buildWizardPrompt(req GenerateRequest, cfg *config.Config) — builds the    │
│ system prompt with available MCP servers, existing entity names, and rules   │
│ - wizardJSONSchema — JSON Schema string constraining Claude's output to Plan │
│  structure                                                                   │
│ - The prompt teaches Claude about all entity types, MCP server tool names,   │
│ naming conventions (kebab-case), and configuration rules                     │
│                                                                              │
│ generator.go — Generator struct:                                             │
│ - NewGenerator(runner *task.Runner) *Generator                               │
│ - Generate(ctx, req, cfg) (*Plan, error) — creates a synthetic config.Task   │
│ with the wizard prompt and json_schema, runs it via task.Runner, parses JSON │
│  output into Plan, assigns UUID + status "draft"                             │
│ - Uses model sonnet for speed, permission_mode: plan (read-only), timeout    │
│ 5m, max_turns 3                                                              │
│                                                                              │
│ applier.go — Applier struct:                                                 │
│ - NewApplier(cfg, subagentMgr, domainMgr) *Applier                           │
│ - Apply(plan *Plan) (*ApplyResult, error) — creates entities in strict       │
│ order:                                                                       │
│   a. Domains → add to cfg.Domains, call domainMgr.AddDomain() (new method)   │
│   b. Agents → subagentMgr.Create() for each                                  │
│   c. Tasks → append to cfg.Tasks                                             │
│   d. Pipelines → append to cfg.Pipelines                                     │
│   e. Save → cfg.Save()                                                       │
│ - Accumulates rollback functions; on any failure, executes rollbacks in      │
│ reverse order                                                                │
│ - Validates before mutating: no duplicate names, MCP servers exist,          │
│ agent/task references valid                                                  │
│                                                                              │
│ store.go — PlanStore (in-memory with 30-min TTL):                            │
│ - NewPlanStore() *PlanStore                                                  │
│ - Put(plan), Get(id) (*Plan, bool), Delete(id)                               │
│                                                                              │
